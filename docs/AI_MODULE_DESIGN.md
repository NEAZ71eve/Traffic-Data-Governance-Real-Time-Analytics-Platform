# AI 辅助模块设计

## 定位与边界

AI 模块作为**辅助工具**，不接入核心数仓/实时链路，可一键关闭不影响核心业务。

```
AI 辅助层: NL2SQL / 异常检测 / ETL 生成
    ↓ 只读查询 / 离线分析 / 开发辅助
核心数仓链路 (ODS→ADS) — 不受 AI 影响
```

## NL2SQL 查询

| 方案 | 部署 | 准确率 | 适用 |
|------|------|--------|------|
| 轻量 (SQLNet/RATSQL) | CPU推理 | 常见~80% | 演示项目 |
| 深度 (ChatGLM+LoRA) | GPU 6GB+ | 复杂~90% | 生产环境 |

**评估目标**:

| 查询类型 | 准确率 |
|---------|--------|
| 单表查询 | ≥90% |
| 多表 JOIN | ≥80% |
| 聚合分析 | ≥75% |
| 复杂子查询 | ≥60% |
| 综合 | ≥85% |

**错误处理**: 解析失败→提示重述 / SQL执行超时→返回部分结果 / 空结果→建议修改条件 / 不支持查询→明确告知

## 异常检测 (Isolation Forest)

| 维度 | 输入 | 检测内容 |
|------|------|---------|
| 车流异常 | 历史车流时序 | 突增/突降/断档 |
| 设备异常 | CPU/内存/温度 | 指标偏离基线 |
| 时序断档 | 上报时间戳 | 数据缺失窗口 |

**安全边界**: 不写数据库 / 不触发重跑 / 不自动修复 / 可独立关闭

## ETL 脚本生成

```python
# 自然语言 → SQL
输入: "创建实时车流表，按分钟汇总"
输出: CREATE TABLE dwd_vehicle_pass_di (...);
     INSERT INTO dwd_vehicle_pass_di
     SELECT window_start, camera_id, COUNT(*) FROM ods_vehicle_pass_di GROUP BY window_start, camera_id;
```

**AI 模块不足与改进**:

| 模块 | 当前不足 | 改进计划 |
|------|---------|---------|
| 异常检测 | 自定义实现，无 scikit-learn，无模型持久化 | 改用 sklearn + 模型持久化 |
| NL2SQL | 规则匹配而非深度学习 | 扩展至15+查询意图 |
| 数据质量 | 未对接真实 Hive/Kafka | 对接真实 Metastore |
| 数据血缘 | 硬编码，无自动解析 | 自动解析 + Web UI |
